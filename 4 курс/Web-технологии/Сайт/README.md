# SportShop — интернет-магазин спортивных товаров

Учебный web-проект для курсовой работы по дисциплине «Web-технологии». Реализован только на **HTML5, CSS3 и JavaScript** без серверной части и без реальной базы данных. Данные хранятся в JSON-файлах и в **localStorage** (корзина, заказы, изменения каталога в админ-панели).

---

## 1. Общая концепция проекта

**Цель:** создать адаптивный сайт интернет-магазина спортивных товаров, в котором:
- **Пользователь** может просматривать каталог, искать и фильтровать товары, открывать карточки товаров, добавлять товары в корзину и оформлять заказ.
- **Администратор** может управлять товарами (добавление, редактирование, удаление) и просматривать заказы с изменением их статуса.

Вся логика выполняется в браузере: загрузка данных из JSON, рендер страниц, фильтрация, корзина, оформление заказа и действия админа сохраняются в **localStorage**. Это имитация работы с данными без сервера.

**Тематика:** 8 категорий — спортивная одежда, обувь, мячи, тренажеры, аксессуары, фитнес, туризм, спортивная экипировка.

## 3. Структура данных JSON

### categories.json
Массив объектов категорий:
```json
[
  {
    "id": "sportswear",
    "name": "Спортивная одежда",
    "slug": "sportswear",
    "description": "Описание категории"
  }
]
```

### products.json
Массив объектов товаров:
```json
[
  {
    "id": "1",
    "name": "Название товара",
    "category": "sportswear",
    "brand": "Nike",
    "price": 2499,
    "oldPrice": 2999,
    "description": "Описание",
    "specs": { "Материал": "Полиэстер", "Размеры": "S, M, L" },
    "image": "https://...",
    "inStock": true,
    "bestseller": false,
    "rating": 4.8
  }
]
```

### localStorage (ключи)
- **sportshop_cart** — корзина: `[{ id, quantity }, ...]`
- **sportshop_orders** — заказы: массив объектов заказа с полями `id`, `items`, `total`, `customer`, `status`, `createdAt`
- **sportshop_products** — при наличии: перезаписанный каталог товаров (после изменений в админке)
- **sportshop_admin_auth** — флаг входа админа (учебная имитация)

## 4. Запуск проекта локально

1. Склонируйте или скопируйте папку проекта на компьютер.
2. Откройте проект в редакторе кода (например, VS Code).
3. Запустите локальный сервер, чтобы избежать ограничений CORS при загрузке JSON через `fetch()`:
   - **VS Code:** установите расширение «Live Server» и нажмите «Go Live» в статус-баре, либо
   - **Node.js:** в корне проекта выполните `npx serve .` или `npx http-server . -p 8080`, либо
   - **Python:** в корне выполните `python -m http.server 8000`.
4. Откройте в браузере адрес, который выдаёт сервер (например, `http://127.0.0.1:5500` или `http://localhost:8080`).
5. Для входа в админ-панель: перейдите по ссылке «Вход для администратора» в подвале, логин **admin**, пароль **admin123**.

**Важно:** при открытии HTML-файлов напрямую через `file://` загрузка JSON может блокироваться браузером. Используйте локальный сервер.

## 5. Краткое описание работы

- **Главная:** загружаются категории и товары из JSON (или из localStorage, если каталог был изменён в админке). Отображаются блоки категорий и популярные товары. Счётчик корзины берётся из localStorage.
- **Каталог:** товары фильтруются по поиску, категории и диапазону цены, сортируются. Карточки рендерятся через JavaScript. Добавление в корзину обновляет localStorage и счётчик.
- **Страница товара:** товар выбирается по `?id=...`. Показываются описание, характеристики, похожие товары из той же категории. Добавление в корзину сохраняется в localStorage.
- **Корзина:** список формируется по данным из localStorage и загруженному каталогу. Можно менять количество и удалять позиции. Итого пересчитывается автоматически.
- **Оформление заказа:** форма с валидацией (ФИО, телефон, email, адрес). После отправки заказ сохраняется в localStorage, корзина очищается, выполняется переход на `success.html?order=...&total=...`.
- **Админ-панель:** вход по фиксированному логину/паролю (учебная имитация). В разделе «Товары» — просмотр, добавление, редактирование и удаление; изменения сохраняются в localStorage и подменяют исходный JSON при следующей загрузке. В разделе «Заказы» — список заказов и смена статуса (новый, подтвержден, отправлен, завершен).

## 6. Какие скриншоты сделать для курсовой

1. Главная страница — hero-блок, категории, блок популярных товаров.
2. Каталог — фильтры (поиск, категория, цена, сортировка) и сетка карточек.
3. Страница товара — изображение, описание, характеристики, кнопка «В корзину», блок «Похожие товары».
4. Корзина — список товаров, изменение количества, итого, кнопка «Оформить заказ».
5. Оформление заказа — форма (ФИО, телефон, email, адрес, комментарий) и блок итога.
6. Страница успешного заказа — сообщение и номер заказа.
7. Страницы «О магазине» и «Контакты».
8. Вход в админ-панель — форма логина (можно указать, что логин/пароль учебные).
9. Админ-панель — вкладка «Товары»: таблица, кнопки «Добавить», «Изменить», «Удалить».
10. Админ-панель — вкладка «Заказы»: список заказов и смена статуса.
11. Адаптивность — вид каталога или главной на мобильном (меню-бургер, сетка товаров).

## 7. Функции для пояснительной записки (реализованные)

- Адаптивная вёрстка (десктоп, планшет, мобильный), мобильное меню.
- Загрузка каталога и категорий из JSON.
- Рендер карточек товаров и страницы товара средствами JavaScript.
- Поиск по названию, фильтр по категории и по диапазону цены, сортировка по цене и названию, сброс фильтров.
- Корзина: добавление, изменение количества, удаление, пересчёт суммы, хранение в localStorage.
- Оформление заказа: форма, валидация обязательных полей (ФИО, телефон, email, адрес), сохранение заказа в localStorage, очистка корзины, переход на страницу успеха.
- Учебная авторизация администратора (фиксированные логин/пароль, флаг в localStorage).
- Админ-панель: просмотр списка товаров, добавление, редактирование и удаление товаров с сохранением в localStorage; просмотр заказов и клиентских данных; изменение статуса заказа.
- Вспомогательные элементы: хлебные крошки, тосты (уведомления), форматирование цен, единый стиль кнопок и форм.

## 8. Разделы курсовой работы, которые можно раскрыть на основе проекта

- **Анализ предметной области** — интернет-магазины, типовой функционал (каталог, корзина, заказ, админка).
- **Постановка задачи** — цели, ограничения (без сервера, только фронтенд), выбор технологий.
- **Разработка информационного обеспечения** — структура JSON (категории, товары), ключи и структура данных в localStorage.
- **Разработка структуры диалога пользователя** — навигация, сценарии: главная → каталог → товар → корзина → оформление → успех; страницы «О магазине», «Контакты».
- **Описание клиентской логики** — загрузка данных, фильтрация, сортировка, корзина, валидация формы, сохранение заказа.
- **Описание административной панели** — вход (учебная имитация), CRUD товаров, работа с заказами и статусами.
- **Тестирование** — проверка сценариев в браузере, проверка адаптивности.
- **Инструкция пользователя** — как открыть сайт, как оформить заказ, как войти в админку (логин/пароль) и управлять товарами и заказами.
